# Follow-up with Service Owners

This workflow template helps you systematically follow up with service owners on outstanding tasks and issues.

## Actions

- [ ] Review current open tasks with stakeholders
- [ ] Follow up with John Smith on authentication service issues
- [ ] Check status of database migration with data team
- [ ] Send summary email to project stakeholders
- [ ] Update JIRA tickets with current status
- [ ] Schedule follow-up meeting if issues remain unresolved

## Reminders

- Remind me tomorrow at 8am to check on progress
- Follow up with service owners weekly
- Alert if any task is blocked for more than 2 days

## Notes

**Service Owner Contacts:**
- Authentication Service: john.smith@company.com
- Database Team: data-team@company.com
- Infrastructure: ops-team@company.com

**Communication Guidelines:**
- Send follow-up emails every Tuesday and Friday
- Escalate to manager if no response after 2 attempts
- Include JIRA ticket references in all communications
- Schedule meetings only if email/chat resolution fails

**Template Usage:**
1. Copy this template for each follow-up cycle
2. Update service owner names and contact info
3. Customize tasks based on current issues
4. Set appropriate due dates and reminders
5. Execute workflow using `conductor workflows -c filename.md`
